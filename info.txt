Modelado PostgreSQL — Lotifyx
Relaciones y Diseño ERD
Marketplace Web Multi-Vendedor — MVP

1. Objetivo del Modelado
El modelo de base de datos de Lotifyx está diseñado para soportar:
marketplace multi-vendedor
escalabilidad modular
subastas en tiempo real
reputación de usuarios
pedidos multi-vendedor
cotizaciones
publicaciones inmobiliarias
trazabilidad y auditoría
futura migración a microservicios
La base de datos utilizará PostgreSQL como motor relacional principal.

2. Principios del Diseño
Arquitectura de datos
El modelo seguirá:
normalización relacional
integridad referencial
separación modular
soft delete
auditoría básica
timestamps estándar

3. Convenciones Generales
Campos estándar
La mayoría de tablas incluirán:
id
created_at
updated_at
deleted_at (soft delete opcional)
status

6. Relaciones Principales del Sistema
Usuarios
Un rol puede tener muchos usuarios.
Un usuario tiene un perfil.
Un usuario puede tener múltiples publicaciones.
Un usuario puede generar múltiples pedidos.
Un usuario puede crear múltiples subastas.
Un usuario puede realizar múltiples pujas.

Productos
Un producto pertenece a una categoría.
Un producto puede tener múltiples imágenes.
Un producto puede tener múltiples variantes.
Un producto puede aparecer en múltiples pedidos.
Un producto puede tener una subasta.

Pedidos
Un pedido pertenece a un comprador.
Un pedido tiene múltiples items.
Cada item pertenece a un vendedor.
Un pedido puede tener un escrow.

Cotizaciones
Una solicitud puede recibir múltiples propuestas.
Un vendedor puede enviar múltiples propuestas.

9. Estrategia de Escalabilidad
PostgreSQL + Redis
PostgreSQL manejará:
relaciones críticas
integridad de negocio
transacciones
auditoría
Redis manejará:
cache
subastas tiempo real
sesiones
colas
rate limiting

10. Estrategia de Soft Delete
Tablas críticas usarán:
deleted_at
status
Beneficios:
recuperación de información
auditoría
historial
prevención de pérdida accidental
